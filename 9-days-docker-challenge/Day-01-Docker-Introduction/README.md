🚀 Day 1 – Introduction to Docker📌 What is Docker?Docker is a containerization platform that allows developers to package applications along with all their dependencies into a single, standardized unit called a container. This ensures absolute consistency across different environments, from a developer's laptop to production servers.❓ Why Docker is Needed?🔴 Before Docker❌ The classic "Works on my machine" problem.❌ Frustrating dependency and version conflicts.❌ Difficult and manual environment setup.❌ Slow, time-consuming deployment cycles.🟢 With Docker✅ Identical environments everywhere (Dev, Test, Prod).✅ Fast, predictable, and reliable deployments.✅ Extremely lightweight compared to traditional Virtual Machines.✅ Effortless horizontal scaling.🆚 Docker vs. Virtual MachineFeatureDocker (Containers)Virtual MachineOSShares the Host OS kernelRequires a separate Guest OSSizeLightweight (Megabytes)Heavy (Gigabytes)Boot TimeMilliseconds to SecondsMinutesPerformanceHigh (Native speed)Moderate (Overhead from Hypervisor)🧱 Docker ArchitectureDocker utilizes a client-server architecture:Docker Client: The CLI where you send commands (e.g., docker run).Docker Daemon (dockerd): The background service that does the heavy lifting (building, running, and managing containers).Docker Images: The read-only blueprints or templates of your application.Docker Containers: The live, running instances of those images.⚙️ Installation & First RunSuccessfully installed Docker on a Linux system.Verify installation:Bashdocker --version
▶️ Running the First Container:Bashdocker run hello-world
📌 What Happens Internally During docker run?The Docker Client tells the Docker Daemon to run the hello-world image.The Daemon checks if the image exists locally.If not found, it pulls the image from the default registry (Docker Hub).The Daemon creates a new container from that image.The container runs the executable, streams the output to the client, and then exits.🧠 Key Concepts LearnedThe fundamental difference between an Image (blueprint) and a Container (process).The core components of Docker's architecture.Basics of the container lifecycle.❗ Issues Faced & ResolvedIssue: Docker service was not running immediately after installation.Solution: Started the daemon manually using systemd:Bashsudo systemctl start docker
